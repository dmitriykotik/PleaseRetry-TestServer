# PleaseRetry-TestServer

**Внимание:** это тестовая версия серверной части — предназначена только для локальной отладки. Не используйте в финальном/продуктовом проекте: код имеет ограничения, упрощения и потенциальные уязвимости. Проект пародирует (эмулирует) поведение Game Coordinator (GC) CS:GO для отладки плагина `PleaseRetry`.

## Структура проекта
```
protobufs/
  csgo.proto        # Proto-файл (описание сообщений)
  csgo_pb2.py       # Скомпилированный тестовый Python-модуль (результат protoc)
protoc/
  protoc.exe        # Компилятор protobuf
build_proto.bat     # Скрипт для компиляции proto из папки protobufs с помощью protoc/protoc.exe
index.py            # Сервер (точка входа)
install.bat         # Устанавливает зависимости из requirements.txt
requirements.txt    # Список Python-зависимостей
run.bat             # Запускает сервер
```

## Описание сервера (index.py)
Сервер слушает TCP-соединения на `IP:PORT` и обрабатывает бинарные сообщения в формате, ожидаемом плагином `PleaseRetry`:

Формат сообщения:
```
uint32 msgType
uint32 msgSize
uint8  payload[msgSize]
```
(младший порядок байт — `struct.unpack('<II', header)` в коде).

Код демонстрирует поведение подстановки: при получении сообщения с `raw_type == 9110` сервер формирует пропатченый ответ и дополнительно отправляет update-пакет с типом 9105.

## Конфигурируемые переменные (в начале index.py)
В начале файла находятся переменные, которые **нужно отредактировать перед запуском**. Ниже — пояснение каждой переменной и допустимые значения.

- `IP` — IP-адрес, на котором будет слушать сервер (по умолчанию `127.0.0.1`).
- `PORT` — TCP-порт для прослушивания (по умолчанию `27010`).

- `ACCOUNT_ID` — целое число (32-битный формат) SteamID (Account ID). **Обязательно** указывать 32-битный формат (не полный 64-битный SteamID). Пример: `116072338`.

### Matchmaking / профиль игрока
- `RANK` — ранг (число 0–18). Значения:
  - 0 — Калибровка
  - 1 — Сильвер 1
  - 2 — Сильвер 2
  - 3 — Сильвер 3
  - 4 — Сильвер 4
  - 5 — Сильвер 5
  - 6 — Сильвер 6
  - 7 — Голд нова 1
  - 8 — Голд нова 2
  - 9 — Голд нова 3
  - 10 — Голд нова 4
  - 11 — Калаш
  - 12 — Калаш с венками
  - 13 — 2 калаша
  - 14 — Биг стар
  - 15 — Беркут
  - 16 — Лем 
  - 17 — Суприм
  - 18 — Глобал

- `WINS` — число выигрышей (матчей) в MM (целое, например `0`, `10` и т.д.).

- `LEVEL` — уровень профиля игрока (1–40).
- `XP` — текущий опыт (например 0–5000 в тесте).

### Наказания / блокировки
- `VAC_BANNED` — флаг VAC-бана (0 или 1).  
  - `0` — нет VAC-бана.
  - `1` — VAC-блокировка (в интерфейсе будет показана желтая плашка VAC lock, если причина = -1 или 0 — см. ниже).

- `PENALTY_REASON` — причина наказания (значения -1, 0, 1.. и далее):
  - `-1` — никаких ограничений/наказаний не отображается.
  - `0` — уведомление об окончании наказания (если `PENALTY_SECONDS == 0` будет показано, что наказание закончено).
  - `1` — Выгнан из прошлой игры (возможно много раз).
  - `2` — Убил много союзников.
  - `3` — Убил союзника в начале раунда.
  - `4` — Не переподключился к матчу.
  - `5` — Бросил прошлую игру (возможно покинул).
  - `6` — Нанёс много урона союзникам.
  - `7` — Нанёс много урона союзникам в начале раунда.
  - `8` — Глобальная блокировка — Ненадёжный.
  - `9` — Слишком часто выгоняли.
  - `10` — Патруль — серьёзное нарушение.
  - `11` — Патруль — малое нарушение.
  - `>=12` — Проверка возможности игры (зелёная плашка / проверка).

- `PENALTY_SECONDS` — время в секундах до окончания наказания (integer). При `PENALTY_REASON == -1` поле игнорируется.

Пояснения по взаимодействию VAC и причины:  
- Если `PENALTY_REASON == -1` — ничего не показывается.  
- Если `PENALTY_REASON == 0` и `PENALTY_SECONDS == 0` — будет уведомление об окончании наказания.  
- Если `VAC_BANNED == 1` и `PENALTY_REASON` равен `-1` или `0`, то отображается желтая плашка VAC-lock. При других значениях причины отображается причина наказания вместо VAC-индикатора.

### Комменданды / лайки (commendation)
- `FRIENDLY` — число лайков "friendly" (целое, например 0–10).
- `TEACHING` — лайки "teaching".
- `LEADER` — лайки "leader".

## Особенности поведения сервера в коде
- Сервер принимает входящие сообщения и по умолчанию отсылает их обратно (echo): `conn.sendall(header + data)`.
- При `raw_type == 9110` код выполняет специальную подстановку: логирует `"[!] Substitution 9110"`, вызывает `patch`, и отправляет два пакета: с типом `9110 | 0x80000000` и с типом `9105 | 0x80000000` (в коде используются установки флагов с `0x80000000` для ответа).

## Установка и запуск
1. Установите Python (рекомендуется Python 3.8+).
2. Запустите `install.bat` чтобы установить зависимости из `requirements.txt` (в Windows):
   ```bat
   install.bat
   ```
   или вручную:
   ```bash
   pip install -r requirements.txt
   ```
3. Скомпилируйте protobuf (если требуется) — запустите `build_proto.bat`. Убедитесь, что `protoc/protoc.exe` доступен. В результате в `protobufs/` должен появиться `csgo_pb2.py` (или замените тестовый, если он уже есть).
4. Запустите сервер:
   ```bat
   run.bat
   ```
   или напрямую:
   ```bash
   py index.py
   ```

## Ограничения и предупреждения
- Этот сервер — тестовая/отладочная утилита. Код НЕ предназначен для использования в продакшне или для распространения в средах, где соблюдение правил платформы/игры важно.
- Сервер не выполняет строгой валидации входящих данных и не шифрует трафик.
- Протокол и отклики сильно упрощены и могут отличаться от реального GC CS:GO.
- Используйте только в локальной тестовой сети.
